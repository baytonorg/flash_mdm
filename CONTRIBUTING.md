# Contributing to Flash MDM

Thanks for considering a contribution to Flash MDM! This document covers what you need to know before submitting a pull request.

## Contributor Licence Agreement (CLA)

Before we can accept your contribution, you must agree to the [Contributor Licence Agreement](./CLA.md). This is required because Flash MDM uses a dual-licensing model (AGPL-3.0 open source + commercial licence), and Jason Bayton / B8N Ltd need the legal right to include your contribution under both.

When you open your first pull request, the CLA Assistant bot will ask you to sign. It's a one-time process - once signed, it covers all future contributions to this project.

**What the CLA does:** It grants Jason Bayton / B8N Ltd a broad licence to use your contribution under any terms, including the commercial licence. You keep full copyright ownership of your code and can use it however you like elsewhere.

**What the CLA does not do:** It does not transfer copyright. You remain the author and copyright holder of your contribution.

## Getting started

1. Fork the repository and create a branch from `main`.
2. Make your changes.
3. If you've added code that should be tested, add tests.
4. Make sure `npm run lint` and `npm run test` pass.
5. Open a pull request.

## Code style

- TypeScript throughout (frontend and backend).
- British English in user-facing strings (authorisation, colour, licence, organisation, etc.).
- Follow the existing patterns in the codebase - if in doubt, look at a similar file.

## Reporting bugs

Open a GitHub issue with:

- A clear description of the problem.
- Steps to reproduce it.
- What you expected to happen vs what actually happened.
- Your environment (browser, Node.js version, OS) if relevant.

## Feature requests

Open a GitHub issue describing the feature, why it's useful, and how you'd expect it to work. Discussion before implementation saves everyone time.
